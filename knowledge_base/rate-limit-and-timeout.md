# 限流与超时问题

## 接口返回 429 Too Many Requests

429 表示请求触发了平台限流。常见触发条件包括：

1. 单个 API Key 每分钟请求次数超过套餐限制。
2. 短时间内并发请求过高。
3. 批处理任务没有做重试间隔。

建议处理方式：

1. 在客户端加入指数退避重试。
2. 降低并发数，避免瞬时流量过高。
3. 将批处理任务拆分为多个小批次。
4. 如果业务量长期超过限制，请升级套餐。

## 请求超时

请求超时通常由网络波动、请求体过大或服务端处理时间过长导致。建议：

1. 检查本地网络和代理配置。
2. 减小单次请求的数据量。
3. 将超时时间设置为 30 到 60 秒。
4. 对失败请求进行有限次数重试。
