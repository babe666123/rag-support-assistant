# API Key 与认证问题

## 接口返回 401 Unauthorized

当接口返回 401 时，通常表示请求没有通过身份认证。请按以下顺序排查：

1. 检查请求头是否包含 `Authorization: Bearer YOUR_API_KEY`。
2. 确认 API Key 没有复制错误、缺少字符或包含多余空格。
3. 确认当前 API Key 没有被删除、禁用或过期。
4. 如果使用测试环境，请确认使用的是测试环境的 API Key。

如果以上步骤都正常，请重新生成一个 API Key 并重试。

## 接口返回 403 Forbidden

403 表示认证通过，但当前账号没有权限访问该资源。常见原因包括：

1. 当前套餐没有开通对应接口。
2. API Key 绑定的应用没有访问该数据源的权限。
3. 管理员限制了该 Key 的可访问 IP。

请联系管理员检查应用权限、套餐权限和 IP 白名单配置。
